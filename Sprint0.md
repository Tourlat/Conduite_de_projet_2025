# Sprint 0

## Objectifs du Sprint 0
- Cahier des charges US
- Infrastructure, technologies
- Répartition des rôles / travail / Backlog

## Cahier des Charges - User Stories

Type de User Stories:
- Nouvelle fonctionnalité
- Correction de bug
- Amélioration de performance
- Refactoring / Amélioration du code

Estimation en points de complexité (1, 2, 3, 4, 5)

### Gestion des Utilisateurs et Projets

US1 : Création de Compte
-   En tant que **nouvel utilisateur**, je veux **créer un compte personnel** afin de **sauvegarder mes informations**.
-   **Type** : Nouvelle fonctionnalité
-   **Priorité** : Haute
-   **Estimation** : 4

US2 : Connexion au Compte
-   En tant qu'**utilisateur enregistré**, je veux **me connecter à mon compte** afin d'**accéder à mes projets et à mon espace de travail**.
-   **Type** : Nouvelle fonctionnalité
-   **Priorité** : Haute
-   **Estimation** : 4

US3 : Gestion de Profil
-   En tant qu'**utilisateur connecté**, je veux **gérer les informations de mon profil** afin de **maintenir mes données à jour et de modifier mon mot de passe**.
-   **Type** : Nouvelle fonctionnalité
-   **Priorité** : Moyenne
-   **Estimation** : 3

US4 : Création de Projet
-   En tant qu'**utilisateur connecté**, je veux **pouvoir créer un nouveau projet** afin d'**organiser mon travail et de commencer à collaborer avec une équipe**.
-   **Type** : Nouvelle fonctionnalité
-   **Priorité** : Haute
-   **Estimation** : 3

US5 : Gestion de Projet
-   En tant que **propriétaire de projet**, je veux **pouvoir gérer les membres et les paramètres du projet** afin de **l'adapter aux besoins de mon équipe et de contrôler les accès**.
-   **Type** : Nouvelle fonctionnalité
-   **Priorité** : Haute
-   **Estimation** : 5

US6 : Sauvegarde des Données
-   En tant qu'**utilisateur**, je veux **que mes données soient sauvegardées automatiquement et de manière sécurisée** afin de **ne pas perdre mon travail en cas de problème**.
-   **Type** : Nouvelle fonctionnalité
-   **Priorité** : Haute
-   **Estimation** : 4

---

### Gestion des Issues et Tâches (Backlog)

US7 : Création d'une Issue
-   En tant que **membre du projet**, je veux **pouvoir créer une issue avec un titre, une description et une priorité** afin de **formaliser un besoin, un bug ou une action à réaliser**.
-   **Type** : Nouvelle fonctionnalité
-   **Priorité** : Haute
-   **Estimation** : 2

US8 : Assignation d'une Issue
-   En tant que **membre d'un projet**, je veux **pouvoir assigner une issue à un membre de l’équipe** afin de **clarifier qui est responsable de sa réalisation**.
-   **Type** : Nouvelle fonctionnalité
-   **Priorité** : Haute
-   **Estimation** : 2

US9 : Modification du Statut d'une Issue
-   En tant que **membre du projet**, je veux **pouvoir modifier le statut d’une issue (ex: À faire, En cours, Terminé)** afin de **suivre son avancement de manière visible pour tous**.
-   **Type** : Nouvelle fonctionnalité
-   **Priorité** : Haute
-   **Estimation** : 2

US10 : Création de Tâches
-   En tant que **membre du projet**, je veux **pouvoir créer des tâches liées à une issue** afin de **découper un besoin complexe en actions atomiques et limitées**.
-   **Type** : Nouvelle fonctionnalité
-   **Priorité** : Haute
-   **Estimation** : 2

US11 : Visualisation du Backlog
-   En tant que **membre du projet**, je veux **visualiser le backlog du projet** afin de **comprendre les tâches à faire et de suivre l'avancement global**.
-   **Type** : Nouvelle fonctionnalité
-   **Priorité** : Haute
-   **Estimation** : 3

US12 : Visualisation des Tâches (Kanban/Liste)
-   En tant que **membre du projet**, je veux **pouvoir visualiser les tâches sous forme de Kanban ou de liste** afin d'**avoir une vue d'ensemble du flux de travail**.
-   **Type** : Nouvelle fonctionnalité
-   **Priorité** : Haute
-   **Estimation** : 2

US13 : Vue des Tâches Assignées
-   En tant que **membre d’un projet**, je veux **pouvoir voir les tâches qui me sont assignées** afin d'**organiser mon travail et de savoir sur quoi me concentrer**.
-   **Type** : Nouvelle fonctionnalité
-   **Priorité** : Moyenne
-   **Estimation** : 2

US14 : Suivi du Temps
-   En tant que **membre du projet**, je veux **pouvoir estimer et enregistrer le temps passé sur une tâche** afin de **suivre l'effort réel et d'améliorer les estimations futures**.
-   **Type** : Nouvelle fonctionnalité
-   **Priorité** : Moyenne
-   **Estimation** : 2

---

### Gestion des Versions (Releases)

US15 : Création de Tags et Releases
-   En tant que **membre de projet**, je veux **pouvoir créer un tag pour marquer une version** afin de **créer des releases et de suivre les déploiements**.
-   **Type** : Nouvelle fonctionnalité
-   **Priorité** : Basse
-   **Estimation** : 2

US16 : Historique des Releases
-   En tant qu’**utilisateur**, je veux **pouvoir visualiser l’historique des releases** afin de **suivre l’évolution du produit et les besoins livrés**.
-   **Type** : Nouvelle fonctionnalité
-   **Priorité** : Basse
-   **Estimation** : 2

---

### Gestion des Tests

US17 : Création de Cas de Test
-   En tant que **testeur**, je veux **pouvoir créer des cas de test liés à des issues ou fonctionnalités** afin de **formaliser les scénarios de validation et garantir la qualité**.
-   **Type** : Nouvelle fonctionnalité
-   **Priorité** : Moyenne
-   **Estimation** : 2

US18 : Exécution d'un Cas de Test
-   En tant que **testeur**, je veux **pouvoir exécuter un cas de test et mettre à jour son résultat (Réussi/Échoué)** afin de **suivre la validation des fonctionnalités**.
-   **Type** : Nouvelle fonctionnalité
-   **Priorité** : Moyenne
-   **Estimation** : 2

US19 : Tableau de Bord des Tests
-   En tant que **membre d'un projet**, je veux **pouvoir visualiser un tableau de bord des tests** afin d'**avoir une vue synthétique de la qualité du produit**.
-   **Type** : Nouvelle fonctionnalité
-   **Priorité** : Moyenne
-   **Estimation** : 2

---

### Gestion de la Documentation

US20 : Création et Édition de Documentation
-   En tant que **membre du projet**, je veux **pouvoir créer et modifier des pages de documentation (en Markdown)** afin de **partager la connaissance au sein de l'équipe**.
-   **Type** : Nouvelle fonctionnalité
-   **Priorité** : Moyenne
-   **Estimation** : 2

US21 : Organisation de la Documentation
-   En tant que **membre du projet**, je veux **pouvoir organiser la documentation en sections ou catégories  (Administrateur, Utilisateur, Développeur)** afin de **rendre l'information facile à trouver et à consulter**.
-   **Type** : Nouvelle fonctionnalité
-   **Priorité** : Moyenne
-   **Estimation** : 2

US22 : Liaison Documentation et Issues
-   En tant que **membre du projet**, je veux **pouvoir relier des pages de documentation à des issues ou des tâches** afin de **donner un contexte aux développeurs**.
-   **Type** : Nouvelle fonctionnalité
-   **Priorité** : Basse
-   **Estimation** : 2


## Infrastructure et technologies
- Documentation: markdown, GitHub Wiki
- Application web
- Langage de programmation: Vue.js (frontend), Spring Boot/Java (backend)
- Base de données: PostgreSQL, Mongodb, run with Docker
- Outils de collaboration: GitHub
- Outils de communication: Slack, Discord
